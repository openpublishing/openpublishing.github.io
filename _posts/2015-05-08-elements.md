Elements are used to represent the parts of your content. For instance, a video, a paragraph, and a blockquote are all elements. Some elements contain other elements (a slideshow can include several pictures). Structure your story by organizing these elements together.

### <a name="section"></a> `Section`

#### Summary

A section is a collection of elements-- even other sections. You can use sections to group similar ideas, insert headings, and annotate regions with audio, geotags, and more.

#### Example

````json
{
  "type": "Section",
  "body": [...]
}
````

#### Fields

| Field | Type | Default | Purpose | Caveats | Version |
| ----- | ---- | ------- | ------- | ------- | ------- |
| `body` | array of [`Element`][1] | `[]` | The elements comprising the section. | Ordering is preserved. | 1 |

### <a name="paragraph"></a> `Paragraph`

#### Summary

Your content is broken down into paragraphs, each consisting of formatted text. Formatting is accomplished using a simple shorthand called [Markdown](#). In this way, you can include strong, ephasized, underlined text, and links.

#### Example

````json
{
  "type": "Paragraph",
  "text": "*Hello* and _welcome_ to the Internet. This is a [link](#) and this is a [mention](http://fb.com/zuck)"
}
````

#### Fields

| Field | Type | Default | Purpose | Caveats | Version |
| ----- | ---- | ------- | ------- | ------- | ------- |
| `text` | `string` | `null` | The formatted content comprising the section. | The text is interpeted as [Markdown](#). | 1 |

### <a name="list"></a> `List`

#### Summary

A list is an ordered (or unordered) collection of Elements.

#### Example

````json
{
  "type": "List",
  "ordered": true,
  "items": [...]
}
````

#### Fields

| Field | Type | Default | Purpose | Caveats | Version |
| ----- | ---- | ------- | ------- | ------- | ------- |
| `items` | array of [`Element`][1] | `[]` | The elements comprising the list. || 1 |
| `ordered` | `boolean` | `false` | A flag indicating whether ordering is significant || 1 |

### <a name="video"></a> `Video`

#### Summary

Any video element.

#### Example

````json
{
  "type": "Video",
  "loop": "off",
  "play": "auto",
  "controls": "hide",
  "source": "http://example.com/movie.mpg" 
}
````

#### Fields

| Field | Type | Default | Purpose | Caveats | Version |
| ----- | ---- | ------- | ------- | ------- | ------- |
| `source` | `string` | --- | The unique URL associated with this story. | Must be a valid URL. | 1 |
| `loop` | [`LOOP_MODE`][2] | `null` | The looping mode to use. || 1 |
| `play` | [`PLAY_MODE`][3] | `null` | The playback mode to use. || 1 |
| `controls` | [`CONTROL_MODE`][4] | `null` | The control scheme to display. || 1 |

| ----- | ---- | ------- | ------- | ------- | ------- |

### <a name="embed"></a> `Embed`

#### Summary

Arbitrary markup that will be rendered in a sandbox.

#### Example

````json
{
  "type": "Embed",
  "mode": "feature",
  "content": "<h1>Hello, World!</h1>"
}
````

#### Fields

| Field | Type | Default | Purpose | Caveats | Version |
| ----- | ---- | ------- | ------- | ------- | ------- |
| `mode` | [`EMBED_MODE`][7] | `null` | The type of embed to display. || 1 |
| `source` | `string` | `null` | The URL to load into the embed. |  Must be a valid URL. | 1 |
| `content` | `string` | `null` | Arbitrary markup to be embedded. | Source or content must be provided. | 1 |

### <a name="slideshow"></a> `Slideshow`

#### Summary

A series is a group of media items that are presented in sequence.

#### Example

````json
{
  "type": "Slideshow",
  "items": [...]
}
````

#### Fields

| Field | Type | Default | Purpose | Caveats | Version |
| ----- | ---- | ------- | ------- | ------- | ------- |
| `items` | array of [`Element`][1] | `[]` | The elements to be shown in sequence. | Ordering is preserved. | 1 |
